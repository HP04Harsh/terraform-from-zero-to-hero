# Terraform: From Zero to Hero 🚀

Welcome to my learning journey! This repository is designed to document my progress in mastering **Terraform**, covering everything from basic syntax to advanced orchestration.

## 📚 Learning Path
- [ ] **Fundamentals:** Providers, Resources, and basic HCL syntax.
- [ ] **State Management:** Understanding `terraform.tfstate`, backends, and locking.
- [ ] **Data Handling:** Variables, Outputs, and Data Sources.
- [ ] **Logic & Control:** Using `count`, `for_each`, and conditional logic.
- [ ] **Modularization:** Creating reusable modules and directory structures.
- [ ] **Advanced:** Provisioners, Workspaces, and Sentinel/Policy as Code.

## 📂 Repository Structure
| Folder | Focus Area |
| :--- | :--- |
| `CountandForLoop` | Implementing iteration and scaling resources. |
| `Data_Types` | Understanding HCL variable types and complex objects. |
| `Variables` | Managing inputs, defaults, and best practices for variables. |
| `Providers` | Configuring and managing cloud provider plugins. |
| `Modules` | Structuring code for reusability and scale. |
| `Lifecycle` | Managing resource creation, destruction, and updates. |
| `Provisioners` | Executing scripts on resources during deployment. |
| `Remote_State` | Storing and locking state files for team collaboration. |
| `Dynamic` & `Dynamic_NSG` | Creating flexible resource blocks using dynamic iterators. |
| `conditions` | Implementing conditional logic in configurations. |
| `terraform_datasource` | Fetching and using external/existing infrastructure data. |
| `Terraform_Import` | Bringing existing manual infrastructure into state. |
| `TIME_Delay` | Managing resource dependencies with `time_sleep`. |
| `Get_Request` | Lab experiments with data retrieval. |

## 🛠 How to Use This Repo
1. **Clone the repo:** `git clone https://github.com/HP04Harsh/terraform_basic_to_advanced.git`
2. **Navigate to a module:** `cd <folder_name>`
3. **Run your tests:**
   ```bash
   terraform init
   terraform plan
   terraform apply
   ```
